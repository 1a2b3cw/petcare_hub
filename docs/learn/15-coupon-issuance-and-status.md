# 第 15 步：优惠券发放与状态管理

## 本次目标

继续完善复购运营模块，把“回访提醒”再往前推进半步，补上优惠券的最小闭环：

- 手动发券
- 查看最近发出的券
- 标记已使用
- 标记已过期

这样运营页就不只是“提醒你该联系谁”，还多了一个真正的运营动作。

## 这一步新增了哪些能力

### 1. 运营页可直接手动发券

现在在 `/operations` 页面里可以直接填写：

- 发给哪位客户
- 券名称
- 优惠类型（立减 / 折扣）
- 优惠值
- 最低消费门槛
- 失效日期
- 备注

然后点击发券，直接写进数据库。

### 2. 最近优惠券列表

运营页会展示最近 12 条优惠券，包含：

- 客户
- 券标题
- 优惠值
- 门槛
- 起止日期
- 当前状态

### 3. 券状态可手动流转

当前最小闭环支持：

- `UNUSED -> USED`
- `UNUSED -> EXPIRED`

也就是说，发出去的券现在可以被运营人员手动标记为：

- 已使用
- 已过期

## 为什么这样做

## 1. 为什么先做手动发券，不做自动发券？

因为自动发券往往会连到更复杂的规则：

- 满足什么条件自动发
- 针对什么客户分层发
- 发券频率怎么限制
- 是否和回访任务联动

这些都很容易把功能做重。

当前阶段先做手动发券更合适，因为它已经能体现：

- 数据模型理解
- 后台运营动作
- 券状态管理

而且足够支持作品演示。

## 2. 为什么优惠券放在运营页，而不是客户详情页？

因为它在业务上更偏“运营动作”，不是客户基础资料。

把它放在运营页有两个好处：

- 回访和发券都集中在同一个工作台
- 页面职责更清楚，后面扩展时不容易乱

客户详情页后面可以考虑做“客户持有券概览”，但当前不急着加。

## 关键文件说明

### `src/lib/validations/coupon.ts`

这里新增了优惠券表单的 Zod 校验：

- `customerId` 必填
- `title` 必填
- `type` 只能是 `CASH` 或 `DISCOUNT`
- `value` 必须大于 0
- `minSpend` 可选，但如果填了必须是数字
- `validUntil` 必填

同时还导出了 `couponTypeOptions`，给表单下拉使用。

### `src/components/operations/coupon-form.tsx`

这里是发券表单组件。

它保持了和前面模块一致的写法：

- 表单组件只负责渲染
- 提交逻辑交给外部 `action`

这种拆法后面如果你想把发券表单放到弹窗里，几乎不用重写。

### `src/app/(dashboard)/operations/actions.ts`

这一轮在这里新增了三类动作：

1. `createCouponAction`
   - 解析表单
   - 校验客户是否存在
   - 解析失效日期
   - 创建优惠券

2. `markCouponUsedAction`
   - 把状态改成 `USED`

3. `markCouponExpiredAction`
   - 把状态改成 `EXPIRED`

并且所有动作都会刷新：

- `/operations`
- `/dashboard`
- `/customers`
- 对应客户详情页

### `src/app/(dashboard)/operations/page.tsx`

运营页现在被拆成三层：

1. 顶部统计卡片
2. 回访任务 + 沉睡客户
3. 发券表单 + 最近优惠券列表

这使得页面开始具备一个真实“门店运营工作台”的感觉，而不是单一列表页。

## 你自己如何复现

如果你想自己重做一遍，建议按这个顺序：

1. 先写 `coupon.ts` 的 Zod schema
2. 再写 `CouponForm`
3. 再补 `operations/actions.ts`
   - 发券
   - 标记已用
   - 标记过期
4. 最后把运营页补上“发券 + 券列表”两个区块

## 验证方式

建议这样测：

1. 打开 `/operations`
2. 给某个客户发一张券
3. 看“未使用优惠券”数字是否增加
4. 看最近优惠券列表是否出现新券
5. 点击“标记已用”
6. 刷新页面，确认状态是否变成“已使用”

## 容易踩坑的点

1. **折扣券和立减券显示逻辑不同**
   当前实现里：
   - `CASH` 显示货币金额
   - `DISCOUNT` 显示 `X 折`

2. **失效日期必须解析成 Date**
   表单传进来的只是字符串，不能直接原样塞进 Prisma。

3. **发券后别忘了刷新运营页**
   否则统计卡片和列表会不一致，看起来像没发成功。
