# 第 18 步：工作台首页打磨

## 本次目标

把首页从“只有几个数字卡片”升级成真正能指导当天工作的后台工作台。

这一步不追求炫技，而是让首页一打开就能回答三个问题：

- 今天有哪些预约要处理？
- 现在最值得点进去的入口是什么？
- 门店经营上当前需要注意什么？

## 我做了什么

这一轮主要修改了：

```text
src/app/(dashboard)/dashboard/page.tsx
```

## 新增了哪些能力

### 1. 顶部操作区

现在首页右上角不只是标题，已经补了两个最常用操作：

- 新建预约
- 处理复购运营

这样首页不再只是看板，而是能直接发起动作。

### 2. 今日预约安排

首页新增了“今日预约安排”表格，会展示当天预约的：

- 时间
- 客户 / 宠物
- 服务
- 员工
- 状态

这让首页开始真正像一个门店当天排班面板。

### 3. 快捷入口

补了四个常用入口：

- 新增客户
- 新增服务
- 新建预约
- 去处理回访

这类入口很适合放在首页，因为后台日常操作通常都很重复。

### 4. 经营提醒

首页现在会直接展示：

- 沉睡客户数量
- 未使用优惠券数量
- 启用中服务数量

这些不是纯“今天的事务”，而是更偏经营健康度的信号。

### 5. 待回访缩略卡片

原来的待回访区块保留，但改成更像首页模块：

- 展示最近几条待回访任务
- 增加“查看全部”入口

这样首页和运营页之间就形成了很自然的跳转关系。

## 为什么这样做

## 1. 为什么先打磨工作台，而不是先做日历？

因为工作台是后台系统的“第一眼”。

用户一登录最先看到的通常不是某个业务列表页，而是首页。  
如果首页太空，就会很像“项目还没做完”。

相反，哪怕你还没做特别复杂的图表或日历，只要首页能真正指导操作，整体完成度就会明显提高。

## 2. 为什么首页要同时放“今日事务”和“经营提醒”？

因为门店后台通常有两种关注点：

1. **今天要做什么**
2. **最近经营情况怎么样**

如果首页只放其中一类，就会显得片面。

所以这一步故意把首页拆成：

- 今日预约安排
- 快捷入口
- 经营提醒
- 回访摘要

这样层次更完整。

## 你自己如何复现

如果你想自己重做这一轮，建议顺序是：

1. 先补查询：把首页真正需要的数据拿出来
2. 再补页面结构：把首页拆成几个职责明确的模块
3. 最后补操作按钮和跳转入口

## 验证方式

可以这样测试：

1. 打开 `/dashboard`
2. 确认顶部统计数字是否正常
3. 看“今日预约安排”是否和预约模块数据一致
4. 点快捷入口，确认能正确跳转
5. 看“经营提醒”的数字是否与运营页一致

## 容易踩坑的点

1. **首页做成纯展示页**
   如果没有快捷入口，首页就还是被动看板，不像真实工作台。

2. **首页只堆数字，不给上下文**
   比如只显示“今日预约 = 4”，但不告诉你是哪 4 条，实际使用价值很低。

3. **把所有东西都塞首页**
   首页应该是摘要，不是完整业务页。  
   所以这一步保留了“查看全部”的跳转，而不是把所有回访、所有运营信息全部塞进首页。
